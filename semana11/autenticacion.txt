El login en una aplicación es el proceso mediante el cual un usuario se identifica con sus credenciales para acceder a ciertas funciones del sistema. Esto permite diferenciar entre usuarios autorizados y personas que no deberían tener acceso a información o acciones internas. Algunas rutas deben estar protegidas porque pueden incluir datos privados, registros de la comunidad o herramientas de administración. Si estas rutas fueran públicas, cualquier persona podría consultar o modificar información sensible. Por eso, la autenticación ayuda a controlar el acceso y mejorar la seguridad del sistema. No deberían exponerse públicamente contraseñas, documentos, teléfonos, direcciones, tokens, claves de API ni ningún dato sensible de la comunidad.
